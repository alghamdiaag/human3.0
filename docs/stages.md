# Human3.0 – Stage Definitions & Gate Conditions

## Overview

The system operates through a fixed sequence of stages.

Each stage has:
- a clear objective
- required outputs
- gate conditions for completion

No stage may be skipped.
No stage may be completed without meeting its gate conditions.

---

## Stage 1: Orientation

### Objective
- prepare the user
- explain the process
- set expectations

### System Actions
- present opening narrative
- explain four dimensions (briefly)
- explain outcome (report + system)
- instruct user to answer honestly

### Required Outputs
- user acknowledgment of process

### Gate Conditions
- user provides initial responses
- user engagement confirmed

---

## Stage 2: Identity Extraction

### Objective
- identify the user’s real operating identity
- detect patterns and contradictions

### System Actions
- ask behavior-based questions
- detect repeated actions
- identify mismatch between stated goals and actions

### Required Outputs
- dominant behavioral pattern
- at least two contradictions
- initial anti-vision signal (what user avoids)

### Gate Conditions
- pattern is clear
- contradictions are identified
- system has behavioral evidence

---

## Stage 3: Dissonance Mapping

### Objective
- locate pressure points in the user’s life
- identify what is not working

### System Actions
- extract areas of frustration
- identify recurring problems
- detect emotional or functional breakdown points

### Required Outputs
- primary area of dissonance
- repeated failure loop
- user acknowledgment of discomfort

### Gate Conditions
- dissonance is clearly defined
- problem is grounded in real examples

---

## Stage 4: Mind Analysis

### Objective
- evaluate mental patterns and perception

### System Actions
- assess focus
- detect overthinking or confusion
- identify default thinking patterns

### Required Outputs
- mental state classification
- key cognitive blockers
- one actionable correction

### Gate Conditions
- at least one contradiction identified
- mental pattern linked to behavior

---

## Stage 5: Body Analysis

### Objective
- evaluate energy and physical capacity

### System Actions
- assess sleep, energy, fatigue
- detect nervous system instability
- identify execution barriers

### Required Outputs
- energy profile
- main physical constraint
- one actionable correction

### Gate Conditions
- physical limitation linked to behavior
- execution impact identified

---

## Stage 6: Spirit Analysis

### Objective
- evaluate meaning and internal alignment

### System Actions
- assess direction and purpose
- detect false goals
- identify internal conflict

### Required Outputs
- meaning gap
- misaligned goal or belief
- one actionable correction

### Gate Conditions
- internal conflict identified
- direction ambiguity clarified

---

## Stage 7: Vocation Analysis

### Objective
- evaluate value creation and external output

### System Actions
- assess current work or direction
- detect lack of value translation
- identify monetization or contribution gaps

### Required Outputs
- current vocation state
- core problem (clarity / execution / distribution / energy)
- initial opportunity direction

### Gate Conditions
- problem categorized clearly
- linked to at least one other dimension

---

## Stage 8: Synthesis

### Objective
- integrate all findings
- identify the true leverage point

### System Actions
- compare all dimensions
- detect cross-dimensional causality
- prioritize intervention

### Required Outputs
- Primary Lever (one dimension)
- Supporting Lever (optional, one only)
- cross-dimension explanation

### Gate Conditions
- only one primary lever selected
- reasoning is consistent across all data

---

## Stage 9: Reorientation

### Objective
- define new direction

### System Actions
- build anti-vision
- build vision MVP
- define one-year direction
- define one-month project

### Required Outputs
- anti-vision statement
- vision MVP
- one-year lens
- one-month project

### Gate Conditions
- vision is realistic (not abstract)
- project is actionable

---

## Stage 10: Execution Design

### Objective
- create a livable system

### System Actions
- define daily levers (2–3 max)
- define weekly structure
- define constraints
- define measurement signals

### Required Outputs
- daily levers
- weekly cadence
- constraints
- 30-day execution plan

### Gate Conditions
- plan is minimal and clear
- actions are executable immediately

---

## Stage Completion Rules

A stage is complete only if:
- required outputs are present
- outputs are consistent
- outputs are grounded in user data

If not:
- system must continue probing
- system must not advance

---

## Failure Handling

If the system detects:
- vague answers
- contradictions without resolution
- missing data

Then:
- return to previous stage
- refine inputs
- rebuild outputs

---

## Final Condition

The system reaches completion only when:

- all stages are validated
- synthesis is coherent
- execution is defined
- report can be generated without gaps

Otherwise:
- system must continue refinement
